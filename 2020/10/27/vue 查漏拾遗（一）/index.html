$options

官网的描述

用于当前 Vue 实例的初始化选项。需要在选项中包含自定义 property 时会有用处。

我在工作中基本上没有用过这个api，现在提出几个疑问：

既然是属性，如何定义？

如何在页面上渲染？

数据是双向绑定的吗？或者说是响应式的吗？

先说答案：半响应式的，被动响应式（我自己起的名字）
解释如下
我们申明：

shooter: '百里守约'

在 created 中更新 做修改：

created() { this.$options.shooter = '后裔' }

页面渲染结果：

射手：后裔
刺客：阿珂

但是，我们去方法中做更新：

changeShooter(){ 
    this.shooter = '马可波罗'
}

页面渲染结果：

射手：后裔
刺客：阿珂

执行方法后，更新并没有渲染到页面上。
又但是，我们继续执行其他响应式数据的修改：

changeAssassin(){
    this.assassin = '孙悟空'
}

页面渲染结果：

射手：马可波罗
刺客：孙悟空

可见在方法中对 $options下的属性修改也生效了。
也就是说$options下属性的更新，并不会去主动触发视图更新；但是会在下一次视图更新时被“附带”更新。
为了避免混乱，我们可以认为 $options 下的属性不是响应式的

结论：

1.通过 $options 可获取在组件顶级声明的额外的属性或者方法（如上 $option.shoot()）
2.$options 下的属性不是响应式的
3.可以用来保存一些需要在视图中使用的常量，比如默认图片、提示语等
